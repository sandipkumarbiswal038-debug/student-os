# Contributing to Student OS Spine

1. Create a branch from `main` using a short name such as `feature/notification-api`.
2. Keep changes limited to one clear task.
3. Run the local backend or frontend checks relevant to your change.
4. Open a pull request and explain which PRD acceptance criteria it satisfies.
5. Request one squad member review before merging.

Never commit passwords, OAuth client secrets, `.env` files, or database credentials.
